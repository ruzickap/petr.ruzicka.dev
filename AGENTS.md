# AGENTS.md

Repo-specific notes for `petr.ruzicka.dev` — a Hugo static "link-in-bio"
site. Generic commit/branch/lint conventions live in the global OpenCode
`AGENTS.md` and are not repeated here.

## What this repo is

- Single-page personal site built with **Hugo** (`minVersion 0.115.0`).
- Theme is **`LucentLink-Hugo`**, pulled in as a git submodule
  (`themes/LucentLink-Hugo`). The README's mention of an "Aerial" theme is
  stale — trust `hugo.toml` and `.gitmodules`.
- There is almost no code. Page content is data in
  `content/_index.md` front matter (profile, `socials`, `links`); the body is
  empty. `layouts/index.html` overrides the theme and wires together theme
  partials (`profile.html`, `link-card.html`, etc.).

## Build & run

- The theme submodule MUST be present or the build fails (the `themes/`
  dir is empty on a plain clone):

  ```bash
  git submodule update --init
  ```

- Local preview: `hugo server --buildDrafts`
- Production build (mirrors CI in `.github/workflows/gh-pages-build.yml`):

  ```bash
  hugo build --minify --printPathWarnings --printI18nWarnings --panicOnWarning
  ```

  `--panicOnWarning` makes Hugo warnings fatal — fix warnings, don't ignore
  them. CI also builds with `timeout-minutes: 5`, so keep the build fast.

## Conventions easy to miss

- `# keep-sorted start` / `# keep-sorted end` blocks (in
  `content/_index.md`, `.mega-linter.yml`, `.lycheeignore`, some workflows)
  must stay alphabetically sorted within each block.
- `CHANGELOG.md` is auto-generated by **release-please** (`release-type:
  simple`) from conventional commit messages. Do not edit it by hand; it is
  excluded from every linter.
- New external URLs are link-checked by `lychee` in CI. Some hosts are
  pre-excluded in `.lycheeignore` (medium, pixabay, unsplash, instagram)
  because they block bots; add a host there only if it reliably fails.

## Linting & CI

- All linting runs through **MegaLinter** (`.mega-linter.yml`); markdown uses
  `rumdl` (config in `.rumdl.toml`, lines wrap at 80, code blocks exempt).
- `mega-linter` only runs on non-`main` branches and skips
  `chore/renovate/*` and `release-please--*` branches.
- HTML in `layouts/` is linted; note the existing `<!-- djlint:off ... -->`
  and `htmlhint-disable-next-line` directives — preserve them when editing.

## Deploy flow

- Push to `main` → GitHub Pages (CNAME `petr.ruzicka.dev`).
- Push to any other branch → Cloudflare Pages preview (`dev` branch).
- The `gh-pages-build` workflow only triggers on changes to `content/`,
  `data/`, `hugo.toml`, `layouts/`, `static/`, or `themes/`.
